## 1. 사이트맵 & 라우팅 (Next.js App Router)

```
/                       랜딩 (비로그인) / 홈 리다이렉트 (로그인)
/signup                 가입 — 이메일 인증
/signup/code            가입 — 추천코드 + 관계 선택
/signup/info            가입 — 기본 정보 + 동의
/signup/vouch           보증 대기 화면 (이메일 인증 불가자)
/onboarding             가입 직후 분기: "소개팅 할래요 / 지인 추천할래요 / 둘 다"
/home                   탐색 피드 (프로필 카드 그리드)
/p/[profileId]          프로필 상세 (+ 관계 경로)
/deck                   내 카드덱 (중매인 대시보드)
/deck/new               지인 프로필 등록 (다단계 폼)
/deck/[profileId]/edit  지인 프로필 수정
/me/profile             내 소개팅 프로필 등록/수정
/likes                  받은 호감 / 보낸 호감 / 제안 (탭)
/matches                성사 목록 (번호 공개)
/leaderboard            이달의 마담뚜 랭킹
/me                     마이페이지 (추천코드 공유, 설정)
/admin                  관리자 대시보드 (통계) — ADMIN 전용
/admin/reports          신고 큐
/admin/flags            자동 감지 큐
/admin/members          회원 관리 (검색·제재)
/admin/vouches          보증 승인
```

- 인증 가드: `/home` 이하 전부 로그인 필수 (NextAuth middleware)
- **하단 탭바 — 역할 적응형(role-adaptive)**
  - 기본(솔로): `홈 🏠 / 호감 💚(뱃지) / 성사 🎉 / MY 👤` — 4탭. **카드덱 탭은 숨김**
  - 덱에 카드 ≥ 1장: `홈 / 카드덱 🃏 / 호감 / 성사 / MY` — 5탭으로 전환
  - 첫 지인 등록 완료 순간 탭이 나타나며 코치마크 1회 표시: "카드덱이 열렸어요 🃏"
  - 판정은 파생 상태 (`내가 owner인 isSelf=false Profile 존재 여부`), 별도 플래그 저장 불필요
- 카드덱 최초 진입점 (탭이 없는 상태): ① 홈 상단 배너 ② MY 메뉴 "지인 추천하고 카드덱 열기" ③ 온보딩 분기

## 2. 페이지별 상세

### 2.1 `/` 랜딩
- 히어로: 서비스 한 줄 소개 + "OO님의 지인의 지인, 어쩌면 인연" 카피
- CTA 2개: [추천코드로 가입하기] [로그인]
- 카톡 인앱 브라우저 대응 필수 (유입 대부분이 카톡 링크)

### 2.2 가입 플로우 (3단계, 단계당 1화면)
```
STEP 1  /signup        이메일 입력 → 6자리 코드 발송 → 입력 검증
                        └ "회사 이메일이 없어요" 링크 → /signup/vouch 경로로 분기
STEP 2  /signup/code   추천코드 입력 → 추천인 이름 확인 카드 표시
                        → "OO님과 어떤 사이인가요?" 관계 칩 선택 (지인/가족/동료/선배/후배/사촌/기타)
STEP 3  /signup/info   이름, 연락처, 생년월일, 직장 + 동의 체크 3종 → 완료
```
- 각 스텝 상단 진행 바 (1/3, 2/3, 3/3)
- STEP 2의 "추천인 확인 카드"가 첫 번째 재미 순간 — 추천인 이름을 크게 보여줘서 "아 맞아, 짱구가 보냈지" 느낌

### 2.3 `/onboarding` 역할 분기
- 카드 3장: 💘 나 소개팅 할래요 → `/me/profile` / 🃏 지인 추천할래요 → `/deck/new` / 🙌 둘 다
- "나중에 하기" 스킵 → `/home`

### 2.4 `/home` 탐색 피드
- 상단: 필터 칩 (성별 자동 / 나이대 / 지역 / 종교) — Phase 3, MVP는 전체 노출
- **솔로 시점 상단 요소 (조건부)**
  - 프로필 완성도 넛지: 본인 프로필 미완성 시 "완성도 70% — 사진·연애관을 채우면 수락률이 올라가요" 카드. **호감 전송은 본인 프로필 완성이 선행 조건** (열람은 자유, 전송 시점에 gate — 프로필 없이 '구경만' 하는 남성 유입을 막는 최소 장치)
  - 지인 추천 배너: 카드덱 미개설 사용자에게 "🃏 소개해줄 지인이 있나요?" 1줄 배너 (닫기 가능, 닫으면 30일 미노출) — 카드덱이 성장 엔진이므로 발견성 확보
- **남성 시점 여성 카드 정보 밀도 강화**: 사진이 실루엣이므로 텍스트가 카드의 전부 — 키워드 칩 2개(직무·취미 대표값) + 추천인 한 줄 소개를 카드에 노출해 "읽고 판단"이 가능하게 함
- 본문: 2열 카드 그리드. 카드 구성 = 대표사진 + 이름·나이 + 지역 + **촌수 뱃지(🔗 2다리)** + 추천인 한 줄 소개 1줄 말줄임
- 노출 제외: 내 프로필, 내가 등록한 지인, 거절 이력 상대, 차단 상대, MATCHED/HIDDEN
- 빈 상태: "아직 프로필이 없어요. 지인을 초대해 보세요" + 추천코드 공유 버튼

### 2.5 `/p/[profileId]` 프로필 상세 ★핵심 화면
```
[사진 캐러셀 — 등록 개수만큼, 인디케이터 도트]
[이름, 나이 · 키 · 지역]
[관계 경로 배너] ← 가장 눈에 띄는 위치, 강조 배경색
  "짱구님의 지인(영희)의 지인(맹구)"  🔗 3다리
[추천인의 한 줄 소개 — 말풍선 스타일]
[기본 정보 그리드: 체형/종교/음주/흡연/돌싱/MBTI]
[직장 · 직무]
[취미 태그 칩]
[이상형 / 연애관]
─────────────────────────
[하단 고정 CTA]
  솔로 열람 시:   [💚 호감 보내기]
  중매인 열람 시: [🤝 내 지인과 연결하기] → 카드덱에서 지인 선택 시트
```
- 호감 전송 시 확인 모달: "호감은 동시에 3명까지만 보낼 수 있어요. 보낼까요?"
- 거절당한/차단 프로필 접근 시 404

### 2.6 `/deck` 카드덱
- 상단 요약: `등록 5 · 진행 중 2 · 성사 1` + 내 마담뚜 랭킹 미니 카드
- 카드 그리드 (2열): 사진 + 이름·나이 + 상태 테두리 컬러 (⬛/🟨/🟩, HIDDEN은 회색 반투명)
- 🟩 성사 카드: "졸업 🎓" 리본 오버레이
- 카드 탭 → 액션 시트: [프로필 보기 / 수정 / 매칭 제안하기 / 숨기기 / 삭제]
- FAB(+): `/deck/new`

### 2.7 `/deck/new` 지인 프로필 등록 (다단계 폼)
```
STEP 1  관계 & 동의: "이 분과 어떤 사이인가요?" 관계 칩
        + ⚠️ "당사자에게 등록 동의를 받았습니다" 체크 필수
        + 당사자 연락처 입력 → 등록 완료 시 통지 발송 안내
STEP 2  기본 정보 (이름/성별/출생년도/키/체형/지역/직장/직무)
STEP 3  라이프스타일 (종교/음주(+주량)/흡연/돌싱/MBTI/취미 칩)
STEP 4  어필 (추천인의 한 줄 소개 ★필수 / 이상형 / 연애관)
STEP 5  사진 (대표 1장 필수, 최대 10장, 드래그 정렬)
```
- 임시저장 지원 (이탈 시 복구)
- `/me/profile`(본인 등록)은 동일 폼에서 STEP 1 생략 + "추천인 한 줄 소개" 대신 "자기 소개"

### 2.8 `/likes` 호감함 (탭 3개)
- **받은 호감**: 보낸 사람 카드 + [수락 💚] [거절] — 거절 탭 시 사유 칩 5종 바텀시트
- **보낸 호감**: 상태 뱃지 (대기 중 D-5 / 수락됨 / 거절됨(사유) / 만료)
- **제안**: 중매인이 보낸 "이 둘 어때?" 카드 (두 프로필 나란히) + 수락/거절
- 여성 우선 정책 반영: 남→여 호감은 여성 수락 전까지 남성 화면에 "전달됨"만 표시

### 2.9 `/matches` 성사
- 성사 카드: 상대 사진 + 이름 + **전화번호 (탭하면 복사/전화)** + 성사일
- "인연 경로 카드 만들기" 버튼 (Phase 3): 경로 이미지 생성 → 카톡 공유
- 중매 성사 건이면 "중매: OO님 🏅" 표기

### 2.10 `/leaderboard` 마담뚜 랭킹
- 월간 탭 / 명예의 전당 탭
- 랭킹 행: 순위 + 이름 + 성사 건수 + 성사율 / 1~3위 왕관 아이콘
- 내 순위 하단 고정 바

### 2.11 `/me` 마이페이지
- **추천코드 공유 카드** (최상단): 코드 + [카톡으로 초대하기] — 성장 엔진이므로 제일 눈에 띄게
- 내 소개팅 프로필 바로가기 / 이름 경로 노출 동의 토글 / 차단 목록 / 로그아웃 / 탈퇴

### 2.12 `/admin` — ADMIN 전용 (middleware gate, 일반 회원에겐 진입점 미노출)
**대시보드 `/admin`**
- 스탯 카드 4종: 전체 회원(주간 증감) / 누적 성사(평균 촌수) / 활성 프로필(대리 등록 비율) / 미처리 신고·감지 건수
- **성비 바** (최상단 상시): 활성 프로필 남녀 비율 + 최근 신규 가입 성비 — 성비 붕괴는 1순위 리스크 지표
- **매칭 퍼널** (최근 30일): 호감 발신 → 사진 공개(여성 수락률) → 최종 성사(남성 확정률)
- 역할 분포: 솔로/중매인/겸용, 중매 성사 비율

**신고 큐 `/admin/reports` · 감지 큐 `/admin/flags`**
- 항목 구성: 출처 태그(🤖 AI 감지 / 🚨 신고) + 사유 라벨(음란성·광고스팸·불법·연락처 노출) + 대상 미리보기 + 점수/신고 수
- 액션: 기각(오탐) / 콘텐츠 삭제 / 프로필 숨김 / 제재 부과(경고·7일 정지·차단) — 처리자·시각 자동 기록
- 자동 비공개(선조치) 항목은 상단 고정 — 기각 시 즉시 재공개되므로 우선 검토

**회원 관리 `/admin/members`**: 검색 / 상세(신고·제재 이력, 추천 관계 트리) / 제재 부과·해제
**보증 승인 `/admin/vouches`**: 대기 목록, 승인/반려

## 3. UI 디자인 가이드

- **무드**: 프라이빗 멤버십 클럽 + 따뜻한 신뢰. 화려한 데이팅 앱 느낌(네온, 그라데이션 과다) 지양
- **컬러**
  - Primary: 트러스트 블루 `#3182F6` (CTA, 링크)
  - 배경: 웜 아이보리 `#FBF6EF` / 카드 서피스: `#FFFFFF`
  - 상태: 성사 그린 `#22C55E` / 진행 옐로 `#FACC15` / 거절·경고 `#EF4444`
  - 관계 경로 배너: 연한 블루 틴트 배경 + 볼드 타이포로 강조
- **타이포**: Pretendard. 본문 16px, 프로필 이름 22px bold, 관계 경로 18px semibold
- **컴포넌트 원칙**: 선택 입력은 전부 칩(chip) — 타이핑 최소화 / 바텀시트 우선(모달 지양) / 터치 타겟 44px+
- **마이크로 인터랙션**: 호감 전송 시 하트 파티클, 성사 시 컨페티, 카드 상태 변경 시 테두리 컬러 트랜지션

## 4. API 라우트 스케치 (App Router Route Handlers)

```
POST /api/auth/email/send        인증코드 발송
POST /api/auth/email/verify      코드 검증
POST /api/auth/referral/check    추천코드 유효성 + 추천인 이름 반환
POST /api/signup                 가입 완료
GET  /api/profiles               피드 (필터/페이지네이션, 제외 규칙 적용)
GET  /api/profiles/[id]          상세 + 관계 경로 포함 (phone 제외)
POST /api/profiles               프로필 생성 (본인/지인)
PATCH /api/profiles/[id]         수정 / 상태 변경
POST /api/likes                  호감 전송 (동시 3건 제한 검증)
PATCH /api/likes/[id]            수락/거절(+사유)  → 수락 시 Match 생성 트랜잭션
POST /api/proposals              중매 제안
PATCH /api/proposals/[id]        수락/거절 → 양측 수락 시 Match 생성
GET  /api/matches                성사 목록 (여기서만 phone 복호화 포함)
GET  /api/leaderboard?month=     랭킹
GET  /api/notifications          알림 목록 / PATCH 읽음 처리
POST /api/reports, /api/blocks   신고/차단
```

- Match 생성은 반드시 트랜잭션: `Like/Proposal 상태 변경 + Match insert + Profile 2건 MATCHED + Notification 2건`
- 크론(Vercel Cron): 매일 1회 `expiresAt < now` PENDING Like → EXPIRED 처리 + 알림

## 5. 개발 착수 순서 (Phase 1 내 세부)

1. Prisma 스키마 반영 + 마이그레이션 + 시드 데이터 (가상의 짱구/철수/영희 네트워크)
2. 가입 플로우 (이메일 인증 → 추천코드 → 정보 입력)
3. 프로필 등록 폼 (본인/지인 공용 컴포넌트) + 사진 업로드
4. 관계 경로 재귀 CTE 서비스 함수 + 단위 테스트 (짱구 예시 4케이스를 테스트 픽스처로)
5. 탐색 피드 + 프로필 상세
6. 호감 → 수락/거절 → Match 트랜잭션 → 번호 공개
7. 인앱 알림 + 마이페이지 (추천코드 공유)
```
